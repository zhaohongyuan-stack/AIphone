## 2. ECS 部署前置条件

### 2.1 服务器要求

- 操作系统：Ubuntu 22.04 LTS 或 CentOS 7+
- 配置：最低 2 核 4G（Java + Python + PG + Redis 四容器同机），推荐 4 核 8G
- 磁盘：40G+（PG 数据 + Redis 持久化 + 日志 + Docker 镜像）
- 网络：需开放公网出口（访问阿里云 CCC / RocketMQ / Beebot / DashScope API）

### 2.2 安全组 / 防火墙端口

| 端口 | 用途 | 是否对公网开放 |
|---|---|---|
| 8080 | Java 后端 REST API | 否（仅内网或加白名单） |
| 8000 | Python IVR API | 否（仅内网或加白名单） |
| 5432 | PostgreSQL | 否（仅 Docker 内网） |
| 6379 | Redis | 否（仅 Docker 内网） |
| 22 | SSH | 是（建议改默认端口 + 密钥登录） |

如果前端/管理后台需从公网访问 Java 或 Python，建议加 Nginx 反向代理 + HTTPS。

### 2.3 软件安装

```bash
# Ubuntu
sudo apt update && sudo apt install -y docker.io docker-compose-plugin git

# CentOS
sudo yum install -y docker docker-compose-plugin git

# 启动 Docker 并设置开机自启
sudo systemctl start docker
sudo systemctl enable docker

# 验证
docker --version
docker compose version
```

### 2.4 阿里云侧准备

在部署前确认以下阿里云资源的白名单/安全组已放通 ECS 公网 IP：

- **RocketMQ 5.x 实例**：ECS 公网 IP 需加入实例的访问白名单
- **CCC 实例**：确认 CCC 回调地址能到达 ECS（如需回调）
- **Beebot 实例**：确认 Agent Key 有效
- **DashScope**：API Key 有效，无 IP 限制

## 3. ECS 部署步骤

### 3.1 拉取代码

```bash
cd /opt
git clone https://github.com/zhaohongyuan-stack/AIphone.git
cd AIphone
```

### 3.2 创建 .env 凭证文件

在项目根目录创建 `.env`（已被 .gitignore 排除，不会提交）：

```bash
cp .env.example .env
vi .env
```

填入真实凭证：

```env
ALIYUN_ACCESS_KEY_ID=你的阿里云AK
ALIYUN_ACCESS_KEY_SECRET=你的阿里云SK
ROCKETMQ_ENDPOINT=rmq-cn-xxxxx.cn-beijing.rmq.aliyuncs.com:8080
ROCKETMQ_INSTANCE_ID=rmq-cn-xxxxx
ROCKETMQ_ACCESS_KEY=你的MQ用户名
ROCKETMQ_ACCESS_SECRET=你的MQ密码
BEEBOT_AGENT_KEY=你的Beebot密钥
DASHSCOPE_API_KEY=sk-你的DashScope密钥
```

### 3.3 首次建表

`schema.sql` 的 `mode` 已改为 `never`，首次部署需手动执行一次建表：

```bash
# 先启动 PostgreSQL 容器
docker compose up -d postgres

# 等待 PG 就绪后执行建表
docker exec -i aiphone-postgres psql -U postgres -d ai_phone < aiphone-src/src/main/resources/schema.sql

# 验证表已创建
docker exec -it aiphone-postgres psql -U postgres -d ai_phone -c "\dt"
```

应看到 `work_order`、`dialogue_detail`、`agent_info` 等表。

### 3.4 确认知识库 Excel 文件存在

docker-compose.yml 挂载了知识库 Excel：

```bash
ls -la IVR/副本知识库导出记录-20260629.xlsx
```

如果文件不存在，从本地上传到 `IVR/` 目录下。此文件用于拒绝解答检测，缺失会导致知识库功能降级（但不影响主链路）。

### 3.5 启动全部服务

```bash
docker compose up -d --build
```

首次构建约 5-10 分钟（Java Maven 下载依赖 + Python pip install）。启动顺序由 docker-compose 的 `depends_on` 保证：PostgreSQL + Redis 先启动 → Java + Python 后启动。

### 3.6 验证容器状态

```bash
docker compose ps
```

四个容器状态应为 `Up`，PG 和 Redis 的 health 状态应为 `healthy`。

## 4. 联调验证

### 4.1 健康检查

```bash
# Java 后端健康
curl http://localhost:8080/actuator/health
# 期望: {"status":"UP"}

# Python IVR API 文档
curl http://localhost:8000/docs
# 期望: 返回 Swagger HTML

# Python 调 Java API 连通性
curl http://localhost:8000/api/agents
# 期望: 返回 {"code":200,"data":[...]}
```

### 4.2 数据库验证

```bash
# 查看 Java 建的表
docker exec -it aiphone-postgres psql -U postgres -d ai_phone -c "\dt"

# 查看是否有坐席数据（需先在 Java 端注册坐席）
docker exec -it aiphone-postgres psql -U postgres -d ai_phone -c "SELECT * FROM agent_info;"
```

### 4.3 Redis 验证

```bash
# 进入 Redis 容器
docker exec -it aiphone-redis redis-cli

# 查看坐席状态缓存
KEYS agent:*
```

### 4.4 真实来电联调

用真实号码拨打 CCC 接入号码，观察：

1. **IVR 接入**：Python 日志应出现 `[CCC事件] Ringing` 和 `[CCC事件] Enqueue`
2. **工单创建**：Python 调 Java API 创建工单，Java 日志应出现工单创建记录
3. **Beebot 对话**：Python 日志应出现 Beebot `begin_session` 和对话内容
4. **转人工**：按键 0 或触发转人工指令后，Python 日志应出现 `[人工队列]` 入队记录
5. **挂机**：Python 日志应出现 `[CCC事件] Released`，工单状态更新

```bash
# 实时查看日志
docker compose logs -f java-backend
docker compose logs -f python-ivr
```

### 4.5 Python→Java API 链路验证

Python 端所有 DB 写操作通过 Java API 完成。如 Java API 不可用，Python 会降级直写库并在日志输出 `降级直接写库` 警告。联调时如看到此警告，检查 Java 容器是否正常运行。

## 5. 已知遗留问题

### 5.1 KB_EXCEL_PATH（暂搁置）

知识库 Excel 路径在 docker-compose.yml 中已挂载为 `/app/data/副本知识库导出记录-20260629.xlsx`，IVR/.env.docker 已配置对应路径。如文件缺失，知识库拒绝解答检测功能不可用，但不影响主通话链路。后期维护时更新 Excel 文件即可。

### 5.2 DashScope API Key 未注入 Java（暂搁置）

Java 端 ASR（语音转文字）功能需要 DashScope API Key。当前 Java Docker 容器已通过环境变量注入 `DASHSCOPE_API_KEY`，但 `application-docker.yml` 中 `aliyun.voice.api-key` 可能未引用。如需 ASR 功能，检查 Java 配置中的引用方式。

### 5.3 schema.sql 首次执行

`mode` 已改为 `never`。如需重建数据库表（如 schema.sql 有更新），手动执行：
```bash
docker exec -i aiphone-postgres psql -U postgres -d ai_phone < aiphone-src/src/main/resources/schema.sql
```

## 6. 日常运维

### 6.1 日志查看

```bash
# Java 日志
docker compose logs -f java-backend

# Python 日志
docker compose logs -f python-ivr

# 全部服务
docker compose logs -f
```

日志持久化在 Docker volume 中（`java-logs`、`python-data`），容器重建后不丢失。

### 6.2 重启服务

```bash
# 重启单个服务
docker compose restart java-backend
docker compose restart python-ivr

# 重启全部
docker compose restart
```

### 6.3 更新代码

```bash
cd /opt/AIphone
git pull origin master

# 如有代码变更需重新构建镜像
docker compose up -d --build java-backend python-ivr
```

### 6.4 数据库备份

```bash
# 导出数据库
docker exec aiphone-postgres pg_dump -U postgres ai_phone > backup_$(date +%Y%m%d).sql

# 恢复数据库
cat backup_xxxxxxxx.sql | docker exec -i aiphone-postgres psql -U postgres -d ai_phone
```

### 6.5 常见故障排查

| 现象 | 排查方向 |
|---|---|
| Java 启动失败 | 检查 PG/Redis 是否 healthy；检查 CCC AK/SK 是否正确 |
| Python 启动失败 | 检查 .env 凭证；检查 RocketMQ endpoint 是否可达 |
| 来电无响应 | 检查 RocketMQ 白名单是否放通 ECS IP；检查 CCC 事件推送配置 |
| 工单未创建 | 查看 Python 日志是否有 Java API 调用失败；检查 Java 容器是否 Up |
| 坐席状态不同步 | 确认两端连同一 Redis（`redis:6379`）；检查 Redis 网络连通性 |
| Beebot 对话失败 | 检查 BEEBOT_AGENT_KEY；确认 Beebot 实例白名单 |

## 7. 关键配置文件索引

| 文件 | 作用 |
|---|---|
| `docker-compose.yml` | 统一容器编排（PG + Redis + Java + Python） |
| `.env` | 敏感凭证（AK/SK、MQ 密钥、API Key），不提交 GitHub |
| `.env.example` | 凭证模板，提交 GitHub |
| `IVR/.env.docker` | Python 非敏感配置（PG/Redis 地址、槽位数等），挂载到容器 |
| `IVR/Dockerfile` | Python 镜像构建 |
| `IVR/core/java_client.py` | Python 调 Java API 客户端单例 |
| `aiphone-src/Dockerfile` | Java 镜像构建 |
| `aiphone-src/src/main/resources/application-docker.yml` | Java Docker profile 配置 |
| `aiphone-src/src/main/resources/schema.sql` | 数据库建表脚本 |
